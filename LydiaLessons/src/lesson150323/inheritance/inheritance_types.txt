1. + Специализация
Дочерний класс - подтип исходного класса (соблюдается принцип подстановки)
2. + Спецификация (abstract)
Принцип подстановки + оставление реализации за потомками
3. - Конструирование
Дочерний класс использует методы родительского, но не является его подтипом, нарушая принцип подстановки
4. - Обобщение
Противоположность специализации, дочерний класс модифицирует и переопределяет методы родительского, становясь более общим
5. + Расширение

6. -/+ Ограничение
Частично запрещаем функциональность при наследовании

7. -/+ Варьирование
Наследник -- вариант того, чем может быть родитель
8. +/- Компинирование
Несколько родителей -- наследуем нужные свойства и получить обобщенное поведение
